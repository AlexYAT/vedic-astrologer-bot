# Snapshot System --- Vedic Astrologer Bot

Документ описывает, как создавать контекстные снапшоты проекта для
работы с ChatGPT при переходе от MVP к продакшну.

------------------------------------------------------------------------

## Зачем нужен snapshot

Snapshot --- это краткая фиксация текущего состояния проекта.

Он позволяет: - начинать новый стратегический чат без потери контекста -
избегать перегруженных длинных диалогов - фиксировать эволюцию
продукта - принимать решения на основе фактов

Один snapshot = одна стратегическая фаза.

------------------------------------------------------------------------

## Когда создавать snapshot

Создавать snapshot обязательно:

-   Перед запуском paywall
-   Перед изменением цены
-   Перед началом нового спринта
-   После внедрения крупного изменения
-   Если чат стал перегруженным

Рекомендуемая частота: 1 раз в неделю.

------------------------------------------------------------------------

только факты - только текущее состояние

------------------------------------------------------------------------

## Как использовать snapshot

1.  Создать новый чат.
2.  Прикрепить snapshot-файл.
3.  Написать:

"Это текущее состояние проекта. Предложи стратегию следующего спринта."

ChatGPT будет работать с чистым структурированным контекстом.

------------------------------------------------------------------------

## Архитектура управления проектом

Чат = генерация решений. Файлы проекта = память и фиксация решений.

Рекомендуемая структура папки product:

product/ │ ├── project_snapshot_YYYY_MM_DD.md ├── roadmap.md ├──
metrics.md ├── product_log.md └── snapshot_instructions.md

------------------------------------------------------------------------

## Главное правило

Никогда не продолжать стратегический диалог, если он длится более 1--2
недель.

Закрыть фазу → создать snapshot → открыть новый чат.

Это предотвращает хаос и ускоряет рост продукта.
